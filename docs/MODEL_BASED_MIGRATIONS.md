# Model-Based Alembic Migrations Guide

This guide explains how to use SQLAlchemy models to automatically generate Alembic migrations instead of manually writing CREATE TABLE statements.

## 🎯 Benefits of Model-Based Migrations

1. **Single Source of Truth**: Models define the schema, migrations are auto-generated
2. **Type Safety**: SQLAlchemy models provide type hints and validation
3. **Maintainability**: Changes to models automatically generate migration diffs
4. **Consistency**: No manual CREATE TABLE statements to maintain
5. **IDE Support**: Full autocomplete and type checking for model fields

## 🔧 Model Examples

### Basic Model Structure

```python
from sqlalchemy import String, Integer, DateTime, Boolean
from sqlalchemy.orm import Mapped, mapped_column
from sqlalchemy.sql import func
from zeroque_common.db.session import Base

class ExampleModel(Base):
    __tablename__ = "example_table"
    
    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    name: Mapped[str] = mapped_column(String(100), nullable=False)
    active: Mapped[bool] = mapped_column(Boolean, default=True)
    created_at: Mapped[DateTime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    updated_at: Mapped[DateTime | None] = mapped_column(DateTime(timezone=True), nullable=True)
```

### Model with Relationships

```python
from sqlalchemy import String, Integer, ForeignKey
from sqlalchemy.orm import Mapped, mapped_column, relationship

class ParentModel(Base):
    __tablename__ = "parents"
    
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    name: Mapped[str] = mapped_column(String(100))
    
    # Relationship
    children: Mapped[list["ChildModel"]] = relationship("ChildModel", back_populates="parent")

class ChildModel(Base):
    __tablename__ = "children"
    
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    parent_id: Mapped[int] = mapped_column(Integer, ForeignKey("parents.id"))
    name: Mapped[str] = mapped_column(String(100))
    
    # Relationship
    parent: Mapped["ParentModel"] = relationship("ParentModel", back_populates="children")
```

## 🚀 Auto-Generating Migrations

### Method 1: Using the Script

```bash
# Generate migration from model changes
python scripts/generate_migration_from_models.py "add timestamp fields to all models"
```

### Method 2: Manual Alembic Commands

```bash
# 1. Import all models (ensure they're registered)
python -c "
import sys
sys.path.append('packages/zeroque_common')
from zeroque_common.models import *
"

# 2. Generate migration
alembic revision --autogenerate -m "description of changes"

# 3. Review the generated migration
cat alembic/versions/[new_revision]_description.py

# 4. Apply the migration
alembic upgrade head
```

## 📋 Migration Workflow

### 1. Making Model Changes

```python
# Before: Basic model
class User(Base):
    __tablename__ = "users"
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    name: Mapped[str] = mapped_column(String(100))

# After: Adding fields
class User(Base):
    __tablename__ = "users"
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    name: Mapped[str] = mapped_column(String(100))
    email: Mapped[str] = mapped_column(String(255), unique=True)  # NEW
    created_at: Mapped[DateTime] = mapped_column(DateTime(timezone=True), server_default=func.now())  # NEW
    updated_at: Mapped[DateTime | None] = mapped_column(DateTime(timezone=True), nullable=True)  # NEW
```

### 2. Generate Migration

```bash
alembic revision --autogenerate -m "add email and timestamps to users"
```

### 3. Review Generated Migration

```python
def upgrade() -> None:
    # ### commands auto generated by Alembic - please adjust! ###
    op.add_column('users', sa.Column('email', sa.String(length=255), nullable=True))
    op.add_column('users', sa.Column('created_at', sa.DateTime(timezone=True), server_default=sa.text('now()'), nullable=False))
    op.add_column('users', sa.Column('updated_at', sa.DateTime(timezone=True), nullable=True))
    op.create_unique_constraint(None, 'users', ['email'])

def downgrade() -> None:
    # ### commands auto generated by Alembic - please adjust! ###
    op.drop_constraint(None, 'users', type_='unique')
    op.drop_column('users', 'updated_at')
    op.drop_column('users', 'created_at')
    op.drop_column('users', 'email')
```

### 4. Apply Migration

```bash
alembic upgrade head
```

## 🛠️ Best Practices

### 1. Always Include Timestamps

```python
class BaseModel(Base):
    """Base model with common fields."""
    __abstract__ = True
    
    created_at: Mapped[DateTime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    updated_at: Mapped[DateTime | None] = mapped_column(DateTime(timezone=True), nullable=True)
```

### 2. Use Proper Indexes

```python
class User(Base):
    __tablename__ = "users"
    
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    email: Mapped[str] = mapped_column(String(255), unique=True, index=True)  # Indexed
    tenant_id: Mapped[str] = mapped_column(String(100), index=True)  # Indexed
    name: Mapped[str] = mapped_column(String(100))  # Not indexed (not frequently queried)
```

### 3. Use Appropriate Data Types

```python
# Good: Specific types
id: Mapped[int] = mapped_column(Integer, primary_key=True)
email: Mapped[str] = mapped_column(String(255), unique=True)
active: Mapped[bool] = mapped_column(Boolean, default=True)
metadata: Mapped[dict] = mapped_column(JSON, default=dict)

# Avoid: Generic types
id = Column(Integer, primary_key=True)  # No type hints
email = Column(String)  # No length specified
```

### 4. Handle Relationships Properly

```python
class Order(Base):
    __tablename__ = "orders"
    
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    user_id: Mapped[int] = mapped_column(Integer, ForeignKey("users.id"))
    
    # Relationship
    user: Mapped["User"] = relationship("User", back_populates="orders")
    items: Mapped[list["OrderItem"]] = relationship("OrderItem", back_populates="order")
```

## 🔍 Troubleshooting

### Common Issues

1. **Models Not Detected**: Ensure all models are imported before running `alembic revision --autogenerate`

2. **Missing Imports**: Add missing imports to model files:
   ```python
   from sqlalchemy import String, Integer, DateTime, Boolean
   from sqlalchemy.orm import Mapped, mapped_column
   from sqlalchemy.sql import func
   ```

3. **Type Errors**: Use proper type hints:
   ```python
   # Good
   created_at: Mapped[DateTime] = mapped_column(DateTime(timezone=True), server_default=func.now())
   
   # Bad
   created_at = Column(DateTime(timezone=True), server_default=func.now())
   ```

### Debugging Auto-Generation

```bash
# Check what Alembic detects
alembic revision --autogenerate -m "debug" --sql

# Compare current database with models
alembic revision --autogenerate -m "compare" --verbose
```
